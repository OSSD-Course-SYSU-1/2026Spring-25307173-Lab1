宇飞天气 (YuFei Weather) — 声明式 ArkTS 天气预报系统


宇飞天气是一款基于鸿蒙（HarmonyOS）系统的纯原生轻量级实时天气应用。项目采用声明式开发范式 ArkUI / ArkTS 编写，通过网络请求实时获取全球天气数据，并拥有高颜值的“沉浸式智能天气主题”与数据驱动的“动态生活指数面板”。



项目功能特
 1. 核心基础功能



多城市天气切换：内置全国主要城市列表（北京、上海、广州等），支持通过原生 `Select` 组件一键切换，极速响应。

实时精准天气：对接国际标准气象网站 API（OpenWeatherMap），动态拉取并展示实时气温、体感温度、环境湿度及精准的天气文本描述。

下拉刷新体验：全界面结合 `Refresh` 与 `Scroll` 组件，支持流畅的下拉阻尼动画，随时随地一键刷新获取最新气象数据。

未来天气预测：支持页面路由传参，一键安全跳转至未来 5 天的天气预报深度页面（Forecast 页）。



###### &#x20;2.  近期核心技术重构与完善（重大升级）



根据实际气象因果关联及前沿 UI/UX 体验，项目近期完成了以下重大代码迭代：


 数据驱动 · 2×3 智能生活指数矩阵：

在主界面下方独创性地添加了一个平滑圆角矩形面板，采用 `Grid` 网格布局实现两行三列设计。格子内容\*\*绝非静态贴图\*\*，而是拥有一套严密、自洽的“气象关联逻辑算法”，根据当天天气实时推算：

1. 适宜穿戴情况：根据气温自动判定（根据当天气温，来划分适宜穿的衣物）。

2. 防晒需求：综合天气现象与紫外线（UV）强度，晴天自动触发“注意防晒”。

3. 室外室内运动：联动空气质量指数（AQI）与雨雪天气，恶劣天气自动建议“宜室内运动”。

4. 是否适宜洗车：基于未来降雨概率趋势，智能预判，防止洗车后突降暴雨。

5. 是否带伞：实时监测当前是否有雨、雪或雷暴现象，决定是否提醒带伞。

6. 是否容易感冒：结合低温环境与天气恶劣程度，动态给出感冒风险提示。





沉浸式智能天气主题（Visual UI Overhaul）：

打破了传统天气 App 死板的白底黑字。系统通过抽取实时天气关键词，自动接管并切换全屏主题色与组件样式，且带有 \*\*500ms 丝滑渐变淡入动画\*\*：

☀️ 晴天：活力浅蓝色系 (`#5E92F3`)

☁️ 多云/阴天：高级质感灰色系 (`#78909C`)

🌧️ 下雨/暴雨：深邃湖蓝色系 (`#0277BD`)

❄️ 下雪：清冷灰蓝色系 (`#90A4AE`)

🌫️ 雾霾/沙尘 ：莫兰迪棕灰色 (`#8D6E63`)




毛玻璃融合卡片设计：

将生活指数网格、预报卡片、详细面板的背景统一重构为带透明度的半透明底色。卡片颜色完美与天气界面主题色融为一体，格线采用微弱半透明白线划分，UI 高级感拉满。


技术栈与架构设计



开发环境：DevEco Studio (支持 HarmonyOS NEXT / API 9+)

开发语言：ArkTS (TypeScript 的超集)

UI 框架：ArkUI 声明式开发范式 (`@Entry`, `@Component`, `@State`, `Grid`, `Refresh`)

网络通信：异步数据拉取与标准 JSON 运行时解析

路由导航：页面间传参及显式状态销毁





📦 项目代码规范优化


在重构过程中，针对 ArkTS 编译器的严格审查进行了深度优化：



1. 规避 build 内部赋值报错：严格遵守声明式 UI 规范，将所有动态颜色获取（`getThemeColors()`）直接绑定于组件属性中，移除了 `build()` 块内的逻辑变量声明。

2. 显式类型推导规范：在城市下拉选择器（`Select`）的配置中，放弃了隐式转换，采用标准的 `let item: SelectOption` 循环构建数组，确保在各版本鸿蒙编译器中均能 100% 一次性编译成功。
